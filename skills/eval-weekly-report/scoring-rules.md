# 评分规则详细配置

## 一、岗位类型判定

### 结果导向岗
- 销售部
- 生产部
- 市场部
- 商务部

**评估重点**：订单数、产量、回款额、客户数、签约金额

### 过程导向岗
- 研发部
- 技术支持部
- 行政部
- 人事部
- 财务部

**评估重点**：里程碑完成、问题解决数、文档交付、流程合规

---

## 二、维度一：成果达成（权重50%）

### 结果导向岗评分标准

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 90-100 | 超额完成，数据亮眼 | "本周签约5家，目标3家，超额67%，回款120万" |
| 75-89 | 按时完成，有具体成果 | "完成客户拜访8家，签约2家，达成目标" |
| 60-74 | 部分完成，有进展 | "拜访客户5家，暂无签约，2家进入报价阶段" |
| 40-59 | 进展缓慢，成果模糊 | "持续跟进客户，维护客户关系" |
| 0-39 | 无实质进展 | "本周无订单" 且无任何主动说明 |

### 过程导向岗评分标准

| 分数 | 标准 | 典型表现 |
|-----|------|---------|
| 90-100 | 里程碑提前完成，质量优秀 | "v2.0功能开发完成，提前2天，通过测试" |
| 75-89 | 按计划推进，有交付物 | "完成3个模块开发，提交PR并合入主分支" |
| 60-74 | 有进展但延期或质量一般 | "完成2个模块，1个延期到下周" |
| 40-59 | 进展缓慢，阻塞较多 | "遇到技术问题，正在研究解决方案" |
| 0-39 | 无实质进展 | "持续开发中" 无任何具体说明 |

## 三、维度二：证据质量（权重30%）

### 证据强度分级

| 等级 | 分数 | 定义 | 示例 |
|-----|------|------|------|
| 强证据 | 90-100 | 外部独立可验证 | 客户签收单、回款截图、第三方平台记录、客户邮件确认 |
| 中证据 | 70-89 | 内部系统可追溯 | Git commit链接、JIRA工单号、内部系统截图、会议纪要路径 |
| 弱证据 | 50-69 | 自述但具体可查 | 详细描述客户名称、具体问题、处理过程（无链接但可核实） |
| 无证据 | 0-49 | 纯文字无法核实 | "完成了客户拜访"、"解决了技术问题" 无任何细节 |

### 证据评分细则

**加分情况：**
- 提供截图/附件：+5分
- 提供外部确认（客户邮件、签收单）：+10分
- 数据有同比/环比对比：+5分

**扣分情况：**
- 声称有文档但未提供路径：-5分
- 路径无法访问：-5分
- 数据口径前后不一致：-10分

## 四、维度三：规划清晰（权重20%）

### 评分标准

| 分数 | 标准 | 示例 |
|-----|------|------|
| 90-100 | 3-5项目标，每项SMART | "1. 周三前完成A客户签约（目标50万）；2. 周五提交月度报告；3. 拜访B/C/D三家客户" |
| 75-89 | 有明确目标，部分量化 | "1. 跟进A客户签约；2. 完成月度报告；3. 拜访3家新客户" |
| 60-74 | 有目标但笼统 | "继续跟进客户，完成报告" |
| 40-59 | 目标模糊 | "推进销售工作，做好本职工作" |
| 0-39 | 无规划或空话 | "全力以赴完成领导交办的任务" |

### SMART检查清单

- **S（具体）**：是否说明具体做什么？
- **M（可衡量）**：是否有数量/金额/完成标准？
- **A（可达成）**：目标是否合理可行？
- **R（相关）**：是否与岗位职责相关？
- **T（时限）**：是否有明确时间节点？

满足5项：90-100分
满足3-4项：75-89分
满足1-2项：60-74分
满足0项：40-59分

## 五、扣分项（红旗信号）

### 空话套话检测

**触发词列表：**
- "持续推进"
- "稳步开展"
- "全力以赴"
- "积极配合"
- "有序进行"
- "不断完善"
- "进一步加强"
- "深入推进"

**规则：**
- 出现1-2次：提醒但不扣分
- 出现3次以上：-10分，触发预警

### 无量化数据

**检测方法：**
扫描全文是否包含数字（不含日期）

**规则：**
- 全文无任何数字指标：-15分，触发预警
- 数字仅出现在日期中：视为无量化，-15分

### 目标未兑现

**检测方法：**
对比上周"下周计划"与本周"完成情况"

**规则：**
- 未兑现且有合理说明：不扣分
- 未兑现且无说明：-10分，触发预警
- 连续2周未兑现：-20分，触发预警

### 文档路径问题

**规则：**
- 声称有文档但无路径：-5分/处
- 累计3处以上：触发预警

## 六、加分项

| 加分项 | 加分 | 条件 |
|-------|------|------|
| 外部验证证据 | +5~10分 | 提供客户确认、第三方平台记录等 |
| 主动风险识别 | +5分 | 识别潜在风险并给出解决方案 |
| 超额完成 | +10分 | 完成率>120%且有数据支撑 |
| 创新贡献 | +5分 | 提出被采纳的改进建议 |

---

## 七、总分计算公式

```
基础分 = 成果达成分×0.5 + 证据质量分×0.3 + 规划清晰分×0.2

总分 = 基础分 + 加分项 - 扣分项

总分上限：100分
总分下限：0分
```

---

## 八、等级判定

| 总分 | 等级 | 绩效系数 | 说明 |
|-----|------|---------|------|
| 90-100 | A | 1.2 | 优秀，超出预期 |
| 75-89 | B | 1.1 | 良好，达到预期 |
| 70-74 | C+ | 1.0 | 合格，基本达标 |
| 60-69 | C | 0.9 | 待改进，存在不足 |
| 0-59 | D | 0.8 | 不合格，需重点关注 |

---

## 九、预警触发条件

满足以下任一条件即触发预警，推送给老板：

1. 总分 < 70分
2. 空话套话≥3次
3. 无任何量化数据
4. 上周目标未兑现且无说明
5. 连续2周同一任务无进展
6. 文档路径问题≥3处
