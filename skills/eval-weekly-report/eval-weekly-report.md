# 员工周报绩效评估 SKILL

员工周报自动评估系统。支持单份评估、批量评估、月度汇总。当用户输入 `/eval`、`/eval-batch`、`/eval-month` 或要求评估周报时触发此SKILL。

## 命令说明

| 命令 | 用途 | 示例 |
|-----|------|------|
| `/eval <文件路径>` | 评估单份周报 | `/eval 刘志波周报.pdf` |
| `/eval-batch <文件夹>` | 批量评估文件夹内所有周报 | `/eval-batch /周报/本周/` |
| `/eval-month <月份>` | 生成月度绩效汇总 | `/eval-month 2025-01` |

## 评估流程

### 步骤1：读取周报
- 支持 Word (.docx) 和 PDF (.pdf) 格式
- 使用 pandoc 或 pdftotext 提取文本内容

### 步骤2：识别岗位类型
根据部门判断岗位类型：
- **结果导向岗**：销售部、生产部、市场部
- **过程导向岗**：研发部、技术支持、行政部、人事部

### 步骤3：按三维度评分

**维度权重：**
- 成果达成：50%
- 证据质量：30%
- 规划清晰：20%

**评分标准参考 `scoring-rules.md` 文件。**

### 步骤4：检查红旗信号

扫描以下问题并记录：
1. 空话套话（"持续推进/稳步开展/全力以赴"等）出现3次以上 → -10分，触发预警
2. 无任何量化数据 → -15分，触发预警
3. 上周目标未兑现且无说明 → -10分，触发预警
4. 连续2周同一任务无进展 → -20分，触发预警
5. 文档路径缺失累计3处以上 → -5分，触发预警

### 步骤5：计算总分
```
总分 = 成果达成分×50% + 证据质量分×30% + 规划清晰分×20% + 加分项 - 扣分项
```

### 步骤6：生成输出

**员工周评报告（DOCX格式）：**
调用 `document-skills:docx` 生成，包含：
- 总分和等级
- 各维度得分明细
- 扣分项说明
- 具体改进建议

**老板预警摘要（仅当触发预警时）：**
- 预警员工列表
- 红旗信号说明
- 一句话总结
- 建议动作选项

## 评分等级对照

| 等级 | 分数 | 说明 |
|-----|------|------|
| A | 90-100 | 优秀 |
| B | 75-89 | 良好 |
| C+ | 70-74 | 合格 |
| C | 60-69 | 待改进 |
| D | <60 | 不合格 |

## 月度汇总规则

执行 `/eval-month` 时：
1. 汇总当月所有周评数据
2. 计算月度平均分
3. 应用调整规则：
   - 趋势调整：持续上升+2分，持续下降-2分
   - 红旗惩罚：月累计红旗≥4次，额外-3分
   - 满勤加分：4周均按时提交+1分
4. 生成：
   - 员工月度绩效单（每人一份）
   - 部门月度汇总表
   - 绩效工资核算表

## 绩效工资系数

| 等级 | 分数 | 系数 |
|-----|------|------|
| A | 90-100 | 1.2 |

| C+ | 70-74 | 1.0 |
| C | 60-69 | 0.9 |

## 注意事项

1. 评估语气：直接客观，不绕弯子
2. 改进建议：必须具体可执行，不说空话
3. 报告格式：使用 `document-skills:docx` 生成DOCX文件
4. 数据存储：评估结果记录到日志，便于月度汇总
